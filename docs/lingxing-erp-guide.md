# 领星 ERP 模块 · 使用文档

本模块把**领星 ERP**（通过其官方 OpenAPI）接入个人工作台，覆盖三类需求：**浏览分析广告/经营数据**、**定时分析并产出优化建议**、以及**受控地对亚马逊广告做写操作**（改预算/竞价、否词、加词等）。所有写操作都是真金白银影响真实店铺，因此整套流程围绕"**数据不够不动手、改动多重把关、可回滚可审计**"设计。

> 一句话安全模型：**密钥只存后端 → 双开关默认全关 → 写操作必须 三重复核 + 确定性护栏 + 人工确认 → 执行前抓回滚快照 → 失败自动熔断 → 全程审计**。

---

## 一、快速开始（开箱即用）

进入侧边栏「**◭ 领星 ERP**」，到 **「配置」** tab，三步即可用：

1. **填 OpenAPI 凭证**：在领星 ERP 后台「开放接口」拿到 `AppID` / `AppSecret`，连同 API 域名（默认 `https://openapi.lingxing.com`）填入。（可选填 MCP key。）→ **保存凭证**
2. **测试连接**：点「测试连接（probe）」，看到 `OpenAPI ✓ 已连通，店铺 N 个` 即成功。
3. **打开总开关**：点「启用数据（只读）」。之后即可在 大盘 / 数据浏览 / 优化引擎 浏览分析。

> 凭证写入后端配置（`data_dir/hub_settings.json`，已 gitignore），不入代码库。AppSecret/MCP key 为只写：已配置后留空表示不修改。

**写操作还需第二把开关**：见「操作执行」tab 的「操作开关」（默认关、带自动失效）。

## 二、各板块（Tab）说明

### 配置
- **① 凭证**：OpenAPI 域名 / AppID / AppSecret（+ 可选 MCP key）。
- **② 测试连接**：调用 probe，验证签名与鉴权是否打通、能读到几个店铺。
- **③ 总开关**：数据总开关。关 = 网关拒绝一切领星调用；开 = 可只读浏览分析。
- **④ 优化参数**（保守默认）：目标 ACOS 系数（×毛利）、单步幅度上限、bid 步长、否词最小点击、冷却天数、分析窗口、**写白名单店铺 SID**。
  - ⚠️ **写白名单为空 = 禁止一切写操作（fail-closed）**。只把你确认要自动优化的店铺 SID 填进去。
- **⑤ 复核模型**：三重复核每一位可选不同模型/智能体（见第四节）。
- **⑥ 优化规则文档**：展示并可编辑广告优化方法论；改后作为 LLM 复核/分析的依据注入。

### 大盘
按店铺 + 时间窗口聚合广告 KPI：花费 / 销售额 / ACOS / ROAS / 订单 / CTR / CVR，含花费-销售-ACOS 趋势图与 Top 活动。**金额按店铺本币显示**。
> 跨店铺**不汇总金额**（¥+£+$ 相加无意义）——单店看完整 KPI，跨店看「全部店铺对比」表（各自本币）。

### 数据浏览
配置驱动的只读数据集：店铺 / FBA库存 / Listing / 产品表现 / 利润 / **SP广告活动·关键词·定向·广告组·搜索词·各类报表 / 投放商品**。可改参数查询、强制刷新；数据走本地缓存（规避限流、秒开）。也用来**查目标 ID 和当前竞价**（手动建工单时要用）。

### 优化引擎（核心）
**确定性规则引擎**，不是 LLM 拍脑袋：
1. 选店铺 + 窗口 → **运行优化引擎**。
2. 目标 ACOS **自动按毛利率推**（盈亏平衡=毛利率，目标=系数×毛利；优先用各活动自己产品的毛利做 per-campaign 目标，取不到回退店铺均值）。
3. 按杠杆列出**候选**（每条带：命中规则 / 支撑指标 / 显著性 / 花费销售ACOS）：
   - **否词**：搜索词高点击 0 单 → 否定
   - **降bid**：高 ACOS（点击足够）→ 新bid = RPC×目标，步长封顶
   - **加bid**：赢家（≥N单且 ACOS≤0.8×目标）→ 小步放量
   - **加预算**：预算打满且盈利 → +X%
   - **收割**：搜索词 ≥N单 → 选目标精准活动/广告组一键加词
4. 每条「**生成工单**」→ 进入「操作执行」的三复核链。

### 自动化建议
每周定时（可配星期/小时）跑分析，产出建议；**只建议不写**。可「立即运行一次」。建议同样可转工单走复核。

### 操作执行（写操作）
- **操作开关**（危险）：开启后进入可写态，带倒计时自动失效；执行失败会**熔断**（自动关开关 + 告警）。
- **从分析运行生成工单** / **＋新建工单**（任意 op 类型：活动预算·启停 / 关键词·定向·广告组 竞价 / 加词 / 否词）。
- **工单详情**：改动 before→after、命中规则、**三重复核结论**（含各自模型）、**确定性护栏逐项**、可**预览请求**（dry-run）。
- **确认执行**（需操作开关开）→ 执行前抓回滚快照 → 执行 → 可**回滚**（改类回到旧值；否词回滚=归档；加词不可一键回滚）。
- **预览/下载报告**：每个工单可生成自包含 HTML 报告（可打印 PDF）：操作前数据 + 优化点 + 为什么 + 三复核 + 护栏。

### 审计
全部领星调用（读/写/探针）的日志：调用方 / 工具 / 类型 / 状态(ok/denied/blocked/error) / 耗时 / 时间，可按状态筛选。

## 三、安全机制（写操作的多重把关）

1. **双开关**：总开关（数据）+ 操作开关（写，带自动失效）。任一关，写操作被拦。
2. **三重独立复核**：每笔写操作由 3 个相互独立、上下文全新的复核员（数据严谨派 / 魔鬼代言人 / 业务平衡派）按方法论审视，**全过且最高风险 < 阈值**才放行；模型不可用 = 失败即拒（fail-closed）。可配不同模型做真异构。
3. **确定性护栏**（代码硬规则，不可被 LLM 绕过）：写白名单、单步幅度上限、**冷却期**（同一对象 N 天不重复动）、bid 上下限、状态合法、否词/加词字段完备。
4. **人工最终确认**：复核+护栏过后，仍需你在 UI 点「确认执行」。
5. **回滚快照 + 熔断**：执行前抓改前状态；执行失败自动关操作开关并告警。
6. **全程审计 + 飞书告警**：执行/失败/回滚/开关变更都留痕、推送。

## 四、复核模型配置（配置 ⑤）

三重复核的每一位可独立选模型：
- **HTTP 文本**：DeepSeek、Apimart(Claude)
- **CLI 智能体**：hermes / claude / codex（走子进程，较慢）
- **自定义模型**：OpenAI 兼容，自填 `base_url` / `model` / `api_key`，引用名 `custom:<id>`

某个不可用会自动回退默认链。**建议把「魔鬼代言人」设成与其它两位不同的模型**，形成真正的多模型异构复核。

## 五、广告优化方法论（配置 ⑥ 可改）

- **数据充分性（底线）**：否词≥15点击且0单；改bid≥15点击；放量≥3单；收割≥3单；窗口 bid/预算14–30天、否词/收割30–90天；剔除最近1–2天（归因延迟）。
- **目标**：盈亏平衡 ACOS = 毛利率；目标 ACOS = 系数×毛利率（默认 0.7）。
- **杠杆公式**：降bid 新bid=RPC×目标ACOS（步长≤15%）；加bid +≤15% 且不超 RPC×目标；预算打满且盈利 +X%；否词 negative exact；收割→精准活动 bid≈RPC×目标 + 原活动否定。
- **顺序**：否词 → 收割 → 调bid → 预算；同对象冷却 7 天。

> 确定性阈值在「④ 优化参数」调；方法论叙述在「⑥ 规则文档」改（注入 LLM 复核/分析）。

## 六、常见问题

- **页面没按钮 / 一直加载**：后端改动需 `systemctl restart ivyea-ops`，前端需浏览器硬刷新（Ctrl+Shift+R）。
- **写操作总被拦**：检查 操作开关是否开、写白名单是否包含该店铺 SID、是否在冷却期、幅度是否超上限。
- **大盘/优化引擎数字为 0**：该账号窗口内可能无广告数据；或目标店铺未投放。
- **货币符号**：按店铺站点本币显示（JP=¥、UK=£、欧洲=€、US=$…）；取不到币种则只显示数字。
- **复核第二模型没生效**：所选 provider 不可用（如 apimart key 无 Claude 文本权限 / CLI 未安装），已自动回退；换一个可用 provider 即可。

## 七、注意事项（如实告知）

- 除「活动预算」外，其余写 op 的真实写回路建议首次使用时走一遍「小改 → 验证 → 回滚」再放手。
- 加词（addKeywords）**不支持一键回滚**（撤销请到领星暂停/归档该词）；否词可回滚（归档）。
- 领星 OpenAPI 文档中**未见"创建广告活动"写接口**，故不支持新建活动。
- 领星接口有**限流**，本地缓存 + 节流已处理，但大范围聚合首次较慢。
